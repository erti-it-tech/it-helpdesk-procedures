# Procédure : Escalade d’un ticket (ITIL Niveau 1 → Niveau 2)

## Critères d’escalade
- Temps dépassé (SLA)  
- Problème hors périmètre N1  
- Problème nécessitant droits admin  
- Problème récurrent  

## Étapes
1. Documenter toutes les actions effectuées  
2. Ajouter captures ou logs  
3. Reclassifier le ticket → N2  
4. Informer l’utilisateur  
5. Transférer au bon groupe support  

## Vérifications
- Le ticket contient un historique complet  
- Le N2 peut prendre la suite sans questions
